# 如何使用个人日记系统

欢迎使用个人日记系统！这个系统让你可以方便地记录技术思考和生活感悟。

## 📖 快速开始

### 1. 选择分类

日记系统分为两个主要分类：

- **技术思考** (`blog/tech/`)：记录编程心得、技术探索、项目总结
- **生活日记** (`blog/life/`)：记录日常生活、读书感悟、成长经历

### 2. 添加新日记

#### 方法A：复制现有文件（推荐）

1. 复制 `blog/tech/index.html` 或 `blog/life/index.html`
2. 重命名为新文件名，如 `2026-04-03-my-first-post.html`
3. 用文本编辑器打开并修改内容
4. 在列表页面添加链接

#### 方法B：使用模板创建

## ✍️ 撰写技巧

### 文章结构建议

```html
<!-- 标题 -->
<h1>文章主标题</h1>

<!-- 元信息 -->
<div class="meta">
    <i class="fas fa-calendar"></i> 发布日期
    <i class="fas fa-tags"></i> 标签
</div>

<!-- 正文 -->
<h2>二级标题</h2>
<p>段落内容...</p>

<h3>三级标题</h3>
<p>更多内容...</p>

<!-- 列表 -->
<ul>
    <li>要点1</li>
    <li>要点2</li>
</ul>

<!-- 代码块 -->
<pre style="background: #f5f5f5; padding: 15px; border-radius: 5px; overflow-x: auto;">
<code>代码内容</code>
</pre>

<!-- 引用 -->
<blockquote>
    引用内容
</blockquote>
```

### 常用样式类

- 高亮文本：`<span style="background: yellow;">高亮</span>`
- 重要提示：`<div style="background: #e3f2fd; padding: 15px; border-radius: 5px;">...</div>`
- 代码高亮：`<code style="background: #f5f5f5; padding: 2px 4px; border-radius: 3px;">...</code>`

## 📤 发布日记

### 使用Git命令行

```bash
# 1. 添加新文件
git add blog/tech/2026-04-03-my-post.html

# 2. 提交更改
git commit -m "添加新日记：文章标题"

# 3. 推送到GitHub
git push origin main
```

### 使用GitHub Desktop

1. 将新文件添加到项目文件夹
2. 打开GitHub Desktop，填写提交信息
3. 点击"Commit to main"
4. 点击"Push origin"

## 🎯 最佳实践

### 1. 文件命名规范

- 使用日期前缀：`2026-04-03-文章标题.html`
- 使用英文或拼音文件名，避免特殊字符
- 保持文件名简洁明了

### 2. 内容组织

- **技术思考**：包含代码示例、技术要点、解决方案
- **生活日记**：记录真实感受、成长经历、读书心得

### 3. 定期更新

- 建议每周至少更新1-2篇
- 保持内容质量和真实性
- 添加适当的标签和分类

### 4. 备份重要内容

- 重要的日记可以本地备份
- 使用Git历史记录查看修改历史

## 🔧 常见问题

### Q: 如何添加图片？

A: 将图片上传到 `files/photos/` 目录，然后在文章中引用：

```html
<img src="../../files/photos/my-image.jpg" alt="图片描述" style="max-width: 100%; border-radius: 10px;">
```

### Q: 如何修改已有日记？

A: 直接编辑对应的HTML文件，然后提交更改：

```bash
git add blog/tech/my-post.html
git commit -m "修改日记内容"
git push origin main
```

### Q: 如何删除日记？

A: 删除文件并提交：

```bash
git rm blog/tech/old-post.html
git commit -m "删除旧日记"
git push origin main
```

### Q: 日记不显示怎么办？

A: 检查：
1. 文件是否在正确的目录
2. HTML语法是否正确
3. 是否已推送到GitHub
4. 等待几分钟让GitHub Pages部署
