# 随笔发布问题调试指南

## 问题现象
发布随笔时提示："发布失败：Invalid request. \"sha\" wasn't supplied."

## 调试步骤

### 步骤 1：打开浏览器开发者工具
1. 打开浏览器（Chrome/Firefox/Safari）
2. 访问 https://xie-l.github.io/
3. 按 F12 打开开发者工具
4. 切换到 "Console"（控制台）标签页
5. 确保 "Info" 级别的日志被显示（通常默认显示）

### 步骤 2：执行发布操作
1. 点击右下角的 "💡" 按钮
2. 选择 "随笔思考" 分类
3. 填写内容（标题可选）
4. 点击 "发布到博客"
5. 观察控制台输出的日志

### 步骤 3：收集日志信息
在控制台中查找以下日志信息：

```
[DEBUG] 准备检查文件: blog/thoughts/YYYY-MM-DD-HHMMSS[-标题].md
[DEBUG] existingFile 结果: {...} 或 null
[DEBUG] 获取到 SHA: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
或
[DEBUG] 文件不存在 (404)，将创建新文件
[DEBUG] 准备调用 createOrUpdateFile，fileSha = ...
[DEBUG] 提供 SHA 参数: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
或
[DEBUG] 未提供 SHA 参数，将创建新文件
[DEBUG] 请求体: {
  "message": "添加随笔: ...",
  "content": "...",
  "sha": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
}
```

### 步骤 4：记录完整的错误信息
如果仍然报错，请记录：
1. 控制台中的所有日志（特别是 [DEBUG] 开头的）
2. 完整的错误消息
3. 网络请求的详细信息：
   - 在开发者工具中切换到 "Network"（网络）标签页
   - 找到失败的请求（通常是红色的）
   - 点击该请求，查看 "Request Payload"（请求负载）

## 常见问题排查

### 问题 1：fileSha 为 null 或 undefined
**现象**：日志显示 `[DEBUG] 准备调用 createOrUpdateFile，fileSha = null`

**原因**：
- `getFileContent` 函数返回的数据结构不正确
- `existingFile.sha` 不存在

**排查方法**：
查看 `[DEBUG] existingFile 结果:` 日志，确认返回的数据结构

### 问题 2：SHA 参数未传递到请求体
**现象**：日志显示 `[DEBUG] 提供 SHA 参数: xxxxx` 但请求体中没有 sha 字段

**原因**：
- `createOrUpdateFile` 函数中的条件判断失败
- `sha` 参数为 `null` 或 `undefined`

**排查方法**：
查看 `[DEBUG] 请求体:` 日志，确认最终的请求体内容

### 问题 3：请求体格式错误
**现象**：日志显示请求体，但格式不正确

**原因**：
- `JSON.stringify` 序列化问题
- 特殊字符未正确处理

**排查方法**：
查看网络请求中的实际请求负载

## 如何提供反馈

请将以下信息发送给我：

1. **完整的控制台日志**（复制所有 [DEBUG] 日志）
2. **错误消息的完整文本**
3. **网络请求的截图**（包括请求负载和响应）
4. **操作步骤**：
   - 是否有标题？
   - 内容是什么？
   - 是否添加了标签？
   - 是否是同一天内的第二次发布？

示例反馈格式：

```
控制台日志：
[DEBUG] 准备检查文件: blog/thoughts/2026-04-10-143025.md
[DEBUG] existingFile 结果: null
[DEBUG] 文件不存在 (404)，将创建新文件
[DEBUG] 准备调用 createOrUpdateFile，fileSha = null
[DEBUG] 未提供 SHA 参数，将创建新文件
[DEBUG] 请求体: {
  "message": "添加随笔: 测试内容",
  "content": "---\ntitle: \"\"\ndate: 2026-04-10T06:30:25.123Z\ncategory: thoughts\n---\n\n测试内容\n"
}

错误消息：
发布失败：Invalid request. "sha" wasn't supplied.

操作步骤：
- 无标题
- 内容："测试内容"
- 无标签
- 是今天的第一次发布
```

有了这些信息，我可以精确定位问题所在。
