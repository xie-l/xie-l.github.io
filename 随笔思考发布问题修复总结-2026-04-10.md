# 随笔思考发布问题修复总结

## ✅ 问题已修复

### 问题描述
用户在 GitHub 个人主页点击右下角的"撰写随笔思考"后，发布失败，报错：
```
Invalid request. "sha" wasn't supplied
```

### 根本原因
通过代码分析发现：

1. **缺少 SHA 参数** ❌
   - 当文件已存在时，GitHub API 要求必须提供 SHA 参数才能更新文件
   - `publishThought` 函数在调用 `createOrUpdateFile` 时没有检查文件是否已存在，也没有提供 SHA 参数

2. **文件名冲突风险** ⚠️
   - 文件名生成逻辑：`${dateStr}.md` 或 `${dateStr}-${slug}.md`
   - 如果同一天发布多个无标题的随笔，文件名会冲突（例如 `2026-04-10.md`）
   - 即使有标题，如果标题前30个字符相同，也会导致文件名冲突

### 修复方案

#### 1. 改进文件名生成逻辑（避免冲突）

```javascript
// 生成文件名（添加时间戳避免冲突）
let filename;
const timeStamp = now.toTimeString().split(' ')[0].replace(/:/g, ''); // HHMMSS格式
if (title) {
    const slug = title.substring(0, 30).replace(/[^a-zA-Z0-9\u4e00-\u9fa5]/g, '-');
    filename = `${dateStr}-${timeStamp}-${slug}.md`;
} else {
    filename = `${dateStr}-${timeStamp}.md`;
}
```

**改进点：**
- 添加时间戳（HHMMSS 格式）到文件名
- 确保每个文件名都是唯一的，避免冲突

#### 2. 添加文件存在性检查和 SHA 获取逻辑

```javascript
// 检查文件是否已存在，如果存在则获取其 SHA
const filePath = `blog/thoughts/${filename}`;
let fileSha = null;
try {
    const existingFile = await getFileContent(filePath);
    if (existingFile && existingFile.sha) {
        fileSha = existingFile.sha;
    }
} catch (error) {
    // 文件不存在是正常现象，继续执行
    if (!error.message.includes('404')) {
        console.warn('检查文件存在性时出错:', error);
    }
}

// 创建或更新文件
await createOrUpdateFile(
    filePath,
    fileContent,
    `添加随笔: ${displayTitle}`,
    fileSha  // 传递 SHA 参数（如果文件已存在）
);
```

**改进点：**
- 在创建/更新文件之前，先检查文件是否已存在
- 如果文件已存在，获取其 SHA 值
- 将 SHA 参数传递给 `createOrUpdateFile` 函数
- 优雅处理 404 错误（文件不存在是正常现象）

### 修复效果
