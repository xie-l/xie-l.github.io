# 后续操作建议

## 1. 创建新文件时

**必须包含正确的 Front Matter**：
```markdown
---
status: published
title: 文章标题
date: YYYY-MM-DD
category: life  # 或 tech, books, analysis, quotes, thoughts
tags: [标签1, 标签2]
---

文章内容...
```

**重要**：文件必须以 `---` 开头，不能有前面的空行！

## 2. 日常同步命令

### 推荐：单向同步（避免冲突）
```bash
# 只在 Obsidian 编辑后同步到 Blog
node scripts/obsidian-sync.js --direction obsidian-to-blog

# 只在 Blog 编辑后同步到 Obsidian
node scripts/obsidian-sync.js --direction blog-to-obsidian
```

### 可选：双向同步（接受冲突）
```bash
# 双向同步，自动处理冲突
node scripts/obsidian-sync.js --direction both

# 查看冲突详情
grep "跳过同步" logs/obsidian-sync-*.log
```

## 3. 处理冲突

如果看到 `target_is_newer` 冲突：

1. **检查冲突文件**：
   ```bash
   node scripts/obsidian-sync.js --direction both 2>&1 | grep "跳过同步"
   ```

2. **判断是否需要处理**：
   - 如果 MD 文件更新 → 保留 MD，无需处理
   - 如果 HTML 文件更新 → 保留 HTML，无需处理
   - 如果两边都修改 → 需要手动合并

3. **手动合并（如果需要）**：
   ```bash
   # 1. 备份两个版本
   cp obsidian-vault/生活日记/xxx.md /tmp/xxx.md.backup
   cp blog/life/xxx.html /tmp/xxx.html.backup
   
   # 2. 手动编辑合并
   code obsidian-vault/生活日记/xxx.md
   
   # 3. 重新同步
   node scripts/obsidian-sync.js --direction obsidian-to-blog --file "生活日记/xxx.md"
   ```

## 4. Git 提交建议

### 查看修改的文件
```bash
git status
```

### 如果只有 index.html 被修改
```bash
# 可以选择不提交索引文件（不推荐，因为索引是博客展示的关键）
# 或者使用 git add -p 选择性地提交
git add -p blog/*/index.html
```

### 推荐：提交所有修改
```bash
git add -A
git commit -m "同步更新: $(date +'%Y-%m-%d %H:%M:%S')"
git push
```

## 5. 故障排查

### 文件未同步
1. 检查 Front Matter 格式
2. 检查文件路径是否正确
3. 检查同步日志：
   ```bash
   node scripts/obsidian-sync.js --direction obsidian-to-blog --file "path/to/file.md" 2>&1
   ```

### 同步失败
1. 检查 Front Matter 是否包含所有必填字段
2. 检查文件编码是否为 UTF-8
3. 检查是否有特殊字符导致解析失败

### 索引未更新
1. 检查索引文件是否存在：`blog/{category}/index.html`
2. 检查是否有写入权限
3. 手动重新生成索引：
   ```bash
   node -e "const { updateCategoryIndex } = require('./scripts/utils/category-index'); updateCategoryIndex('./blog', 'life', null, null, [], '', '').then(console.log)"
   ```

## 6. 最佳实践

### 写作流程
1. **在 Obsidian 中创建文件**：确保 Front Matter 正确
2. **编辑内容**：随时保存
3. **同步到 Blog**：运行 `node scripts/obsidian-sync.js --direction obsidian-to-blog`
4. **检查博客**：访问 `https://xie-l.github.io/blog/{category}/` 查看

### 定期维护
1. **每周同步一次**：确保所有文件同步
2. **检查冲突**：查看是否有需要手动处理的冲突
3. **备份重要文件**：定期备份 obsidian-vault 文件夹

### 避免问题
1. **不要手动修改 HTML 文件**：所有修改应在 MD 文件中进行
2. **保持 Front Matter 完整**：不要删除或修改 Front Matter 字段
3. **使用标准日期格式**：`YYYY-MM-DD`
4. **避免特殊字符**：在 Front Matter 中使用标准字符

## 7. 快速参考

### 常用命令
```bash
# 同步所有文件（Obsidian → Blog）
node scripts/obsidian-sync.js --direction obsidian-to-blog

# 同步所有文件（Blog → Obsidian）
node scripts/obsidian-sync.js --direction blog-to-obsidian

# 双向同步
node scripts/obsidian-sync.js --direction both

# 同步单个文件
node scripts/obsidian-sync.js --direction obsidian-to-blog --file "生活日记/文件名.md"

# 查看同步日志
tail -f logs/obsidian-sync-*.log

# 检查特定分类
ls -lt blog/life/*.html | head -10
```

### 分类映射
```
Obsidian 文件夹 → Blog 分类
├── 生活日记 → life
├── 技术思考 → tech
├── 书籍阅读 → books
├── 数据分析 → analysis
├── 摘录记录 → quotes
└── 随笔思考 → thoughts
```

## 8. 总结

✅ **文件格式**：确保 Front Matter 正确，以 `---` 开头
✅ **同步命令**：使用单向同步避免冲突，或接受双向同步的冲突
✅ **冲突处理**：`target_is_newer` 是正常保护机制，无需处理
✅ **Git 提交**：定期提交，保持仓库同步

**记住**：系统已经智能处理文件比较，只同步需要更新的文件。你只需关注内容创作！
