**错误 URL**: `blog/thoughts/2026-04-10（202604）.html`

通过分析发现，系统中有**两个不同的发布入口**：

1. **管理后台** (`admin/main.js` - `publishThought` 函数)
   - 生成 `.md` 文件
   - 已修复（添加 SHA 参数）
   - ✅ 修复状态：已完成

2. **主页右下角快速发布** (`index.html` - 第 2370 行)
   - 生成 `.html` 文件
   - **未修复** - 这是问题的真正来源
   - ❌ 问题状态：未提供 SHA 参数

## 🔧 修复方案

### 修复文件：`index.html`

在发布前添加文件存在性检查，获取 SHA 参数：

```javascript
try {
    const apiBase = 'https://api.github.com/repos/' + owner + '/' + repo + '/contents/';
    
    // 检查文件是否已存在，获取 SHA
    let fileSha = null;
    try {
        const checkResp = await fetch(apiBase + encodeURIComponent(path), {
            method: 'GET',
            headers: {'Authorization':'token '+token}
        });
        if (checkResp.ok) {
            const fileData = await checkResp.json();
            fileSha = fileData.sha;
        }
    } catch(e) {
        // 文件不存在是正常现象
    }
    
    const body = { message:'发布博客：'+title, content:encoded };
    if (fileSha) {
        body.sha = fileSha; // 更新时需要提供 SHA
    }
    
    const resp = await fetch(apiBase + encodeURIComponent(path), {
        method: 'PUT',
        headers: {'Authorization':'token '+token,'Content-Type':'application/json'},
        body: JSON.stringify(body)
    });
    // ...
}
```

### 修复逻辑

1. **文件存在性检查**
   - 使用 GET 请求检查文件是否已存在
   - 如果存在，获取文件的 SHA 值
   - 如果文件不存在（404），继续执行

2. **SHA 参数传递**
   - 创建请求体时，如果 fileSha 有值，添加到 body 中
   - GitHub API 要求更新已存在文件时必须提供 SHA

3. **错误处理**
   - 优雅处理文件不存在的情况（404 是正常现象）
   - 其他错误会抛出并显示给用户
