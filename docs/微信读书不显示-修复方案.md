## 方案B：完整配置（12分钟）

### 步骤1：获取微信读书Cookie

**操作说明**：

1. 打开浏览器，访问 https://weread.qq.com
2. 扫码登录微信读书
3. 按F12打开开发者工具
4. 切换到 **Application** 标签
5. 左侧选择 **Cookies → weread.qq.com**
6. 找到 **wr_skey** 这个Cookie
7. 复制 **Value** 列的值（一长串字符）

**截图示例**：
```
Application → Cookies → weread.qq.com
┌──────────┬──────────────────────────────┐
│ Name     │ Value                        │
├──────────┼──────────────────────────────┤
│ wr_skey  │ abcdef1234567890abcdef123... │
└──────────┴──────────────────────────────┘
                        ↑
                    复制这里
```

### 步骤2：配置GitHub Secret

**操作说明**：

1. 访问：https://github.com/xie-l/xie-l.github.io/settings/secrets/actions
2. 检查是否存在 **WEREAD_COOKIE**
   - **如果存在**：
     - 点击 **Update**
     - 粘贴新的Cookie值
     - 点击 **Save**
   - **如果不存在**：
     - 点击 **New repository secret**
     - **Name**: `WEREAD_COOKIE`
     - **Value**: 粘贴Cookie值
     - 点击 **Add secret**

**截图示例**：
```
Settings → Secrets and variables → Actions
┌─────────────────────────────────────────┐
│ Repository secrets                      │
│                                         │
│ WEREAD_COOKIE  ●●●●●●●●●●●●●●●●  [Update]│
│                                         │
│ [New repository secret]                 │
└─────────────────────────────────────────┘
```

### 步骤3：触发GitHub Actions工作流

**操作说明**：

1. 访问：https://github.com/xie-l/xie-l.github.io/actions/workflows/fetch-weread.yml
2. 点击 **Run workflow** 按钮
3. 在弹出的下拉菜单中：
   - Branch: `main`（默认）
   - 点击 **Run workflow**
4. 等待运行完成（约1-2分钟）
5. 查看日志确认成功

## 四、故障排查

### 问题1：配置了Cookie但数据仍为空

**可能原因**：
1. Cookie值错误（未复制完整）
2. Cookie已过期（重新获取）
3. 微信读书账号无阅读数据
4. API接口变更

**解决方案**：
- 重新获取Cookie
- 检查工作流日志
- 本地测试API

### 问题2：工作流运行失败

**可能原因**：
1. 网络问题
2. API接口变更
3. 脚本错误

**解决方案**：
- 查看工作流日志
- 检查错误信息
- 更新脚本

### 问题3：数据文件格式错误

**可能原因**：
1. JSON格式错误
2. 编码问题

**解决方案**：
- 验证JSON格式
- 检查文件编码（UTF-8）
