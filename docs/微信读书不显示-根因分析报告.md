# 微信读书不显示问题 - 根因分析报告

**日期**：2026年4月8日
**问题**：主页右侧微信读书统计栏未显示
**根本原因**：weread.json数据为空，导致widget保持display:none
**状态**：✅ 根因已定位

---

## 一、问题现象

### 用户反馈
> "主页右侧没出现微信阅读这一栏"

### 系统行为
- ✅ HTML结构存在：`<div id="weread-widget" style="display:none">`
- ✅ JavaScript代码存在：`loadWeRead()`函数完整
- ✅ CSS样式正常：无特殊隐藏样式
- ❌ **Widget未显示**：保持`display:none`状态

---

## 二、根因分析

### 2.1 代码逻辑追踪

**JavaScript代码**（index.html 第1616行）：
```javascript
if (!d.reading && (!d.recent || d.recent.length === 0)) return;  // 无数据，不显示
```

**执行流程**：
```
loadWeRead() 被调用
    ↓
fetch('/data/weread.json')
    ↓
获取数据：{"reading": null, "recent": []}
    ↓
检查条件：!null && ![] → true && true → true
    ↓
执行 return（提前退出函数）
    ↓
widget.style.display = ''  **未执行**
    ↓
widget保持 display:none
    ↓
**用户看不到widget**
```

### 2.2 数据验证

**远程数据文件**：
```bash
$ curl -s https://xie-l.github.io/data/weread.json
{
  "reading": null,
  "recent": [],
  "updated": "2026-04-08 15:22 UTC"
}
```

**数据状态**：
- `reading: null`（没有正在读的书）
- `recent: []`（最近读过的书列表为空）
- `updated: 2026-04-08 15:22 UTC`（最后更新时间）

**结论**：数据文件为空，满足隐藏条件

### 2.3 为什么数据为空

**数据抓取流程**：
```
GitHub Actions (每天 10:05 UTC)
    ↓
读取 WEREAD_COOKIE (GitHub Secret)
    ↓
调用微信读书API (weread.qq.com)
    ↓
写入 data/weread.json
    ↓
GitHub Pages 部署
```

**可能原因**：
1. **WEREAD_COOKIE 未配置**（最可能）
2. **Cookie 已过期**（有效期7-30天）
3. **微信读书账号无阅读数据**
4. **API接口变更**

**证据**：
- 工作流运行成功（无错误日志）
- 数据文件为空（无error字段）
- 说明工作流读取了cookie，但cookie值为空或无效

## 四、解决方案

### 方案1：配置WEREAD_COOKIE（推荐）

**步骤**：
1. 获取微信读书Cookie（wr_skey）
2. 配置到GitHub Secrets（WEREAD_COOKIE）
3. 触发GitHub Actions工作流
4. 等待数据更新

**优点**：
- 自动更新，无需手动干预
- 数据实时同步

**缺点**：
- 需要定期更新Cookie（7-30天）

### 方案2：手动创建测试数据（快速验证）

**步骤**：
1. 编辑data/weread.json
2. 添加测试数据
3. 提交并推送
4. 验证显示

**优点**：
- 快速验证功能是否正常
- 无需配置Cookie

**缺点**：
- 数据不会自动更新
- 仅用于测试

### 方案3：添加默认提示

**修改JavaScript**：
```javascript
// 修改前
if (!d.reading && (!d.recent || d.recent.length === 0)) return;

// 修改后
if (!d.reading && (!d.recent || d.recent.length === 0)) {
    // 显示提示信息
    const content = document.getElementById('weread-content');
    if (content) {
        content.innerHTML = '<div style="padding:20px;text-align:center;color:var(--text-light);">暂无阅读数据<br><small style="font-size:12px;">配置Cookie后自动同步</small></div>';
    }
    return;
}
```

**优点**：
- 用户知道功能存在
- 提示如何配置

**缺点**：
- 需要修改代码
