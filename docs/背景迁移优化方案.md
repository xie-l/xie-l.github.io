# 主页背景优化方案

**日期**：2026年4月8日
**目标**：优化index.html的Three.js动态背景效果

---

## 一、现状分析

### ✅ 已实现的功能

index.html已经具备完整的Three.js动态背景：

1. **HTML结构**
   - 包含`<canvas id="hero-canvas"></canvas>`
   - 包含CSS回退背景`.banner-bg-fallback`

2. **CSS样式**
   - hero-canvas: `position: absolute; z-index: -1;`
   - banner-bg-fallback: `position: absolute; z-index: -2;`
   - 渐变背景：linear-gradient(135deg, var(--hero-bg) 0%, var(--hero-mid) 55%, var(--hero-end) 100%)

3. **JavaScript实现**
   - `initThreeJSBackground()`函数完整
   - 3000个粒子系统
   - 自动旋转动画
   - WebGL支持检测
   - 失败回退机制

### ❌ 存在的问题

1. **粒子效果不明显**
   - 粒子数量：3000个（about-new.html有5000个）
   - 粒子大小：0.005（可能太小）
   - 透明度：0.6（可能不够明显）

2. **颜色对比度不足**
   - 粒子颜色：#64ffda（青绿色）
   - 背景也是深蓝色系，对比度不够

3. **动画效果单一**
   - 只有旋转，没有鼠标交互
   - 没有景深变化

---

## 二、优化方案

### 方案A：增强粒子效果（推荐）

**修改内容**：

1. **增加粒子数量**
   ```javascript
   // 从3000增加到5000
   const particlesCount = 5000;
   ```

2. **增大粒子尺寸**
   ```javascript
   // 从0.005增加到0.008
   size: 0.008,
   ```

3. **提高亮度和对比度**
   ```javascript
   // 颜色改为更亮的青色
   color: '#00ffff',
   // 透明度提高到0.8
   opacity: 0.8,
   ```

4. **添加鼠标交互效果**
   ```javascript
   // 添加鼠标移动影响粒子旋转
   let mouseX = 0, mouseY = 0;
   document.addEventListener('mousemove', (e) => {
       mouseX = (e.clientX / window.innerWidth) * 2 - 1;
       mouseY = -(e.clientY / window.innerHeight) * 2 + 1;
   });
   
   // 在animate函数中添加
   particles.rotation.x += 0.0005 + mouseY * 0.0001;
   particles.rotation.y += 0.0005 + mouseX * 0.0001;
   ```

**优点**：
- 效果明显提升
- 增加交互性
- 保持性能优化（像素比限制、requestAnimationFrame）

**缺点**：
- 增加2000个粒子，GPU负担略增
- 低性能设备可能卡顿

---

### 方案B：完全迁移about-new.html的背景

**修改内容**：

1. **复制about-new.html的粒子参数**
   - 粒子数量：5000
   - 粒子大小：0.005
   - 颜色：#64ffda
   - 添加景深模糊效果

2. **迁移CSS变量**
   ```css
   :root {
       --primary-900: #0a192f;
       --primary-800: #112240;
       --primary-700: #233554;
       --secondary: #64ffda;
   }
   ```

**优点**：
- 效果与about-new.html完全一致
- 已经过验证

**缺点**：
- 需要大量代码修改
- 可能引入about-new.html的特定依赖

---

### 方案C：最小化调整

**修改内容**：

仅调整3个参数：
1. 粒子数量：3000 → 4000
2. 透明度：0.6 → 0.75
3. 粒子颜色：#64ffda → #00ffff

**优点**：
- 改动最小，风险最低
- 效果提升明显

**缺点**：
- 效果提升有限

---

## 三、推荐方案：方案A

**理由**：
1. 平衡效果提升和代码改动量
2. 增加交互性，用户体验更好
3. 保持现有架构，不引入新依赖
4. 性能影响可控（仅增加2000粒子）
