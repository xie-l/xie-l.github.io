## 二、根本原因分析

### 数据抓取流程

```
GitHub Actions (每天 10:05 UTC)
    ↓
读取 WEREAD_COOKIE (GitHub Secret)
    ↓
调用微信读书API (weread.qq.com)
    ↓
写入 data/weread.json
    ↓
GitHub Pages 部署
    ↓
主页加载并显示
```

### 失败点

在第二步：**读取 WEREAD_COOKIE**

**可能情况**：
1. **GitHub Secret未配置**：WEREAD_COOKIE不存在
2. **Secret值为空**：WEREAD_COOKIE存在但值为空字符串
3. **Cookie已过期**：有效期7-30天，需要重新获取

**工作流脚本证据**（fetch-weread.yml 第37-44行）：
```python
if not cookie:
    print('[WARN] WEREAD_COOKIE not set, writing empty placeholder.')
    result = {'reading': None, 'recent': [], 'updated': now_utc, 'error': 'WEREAD_COOKIE 未配置'}
    ...
    exit(0)
```

**关键点**：
- 如果cookie未配置，会写入包含error字段的数据
- 当前数据没有error字段，说明cookie变量有值（可能是空字符串）
- 但API返回空数据或调用失败

## 三、解决方案

### 方案：配置或更新 WEREAD_COOKIE

**操作步骤**：

#### 步骤1：获取微信读书Cookie

1. 打开浏览器，访问 https://weread.qq.com
2. 扫码登录微信读书
3. 按F12打开开发者工具
4. 切换到 **Application** 标签
5. 左侧选择 **Cookies → weread.qq.com**
6. 找到 **wr_skey** 这个Cookie
7. 复制 **Value** 列的值（一长串字符）

**截图说明**：
```
[Application] → [Cookies] → [weread.qq.com]
┌──────────┬─────────────┬──────────┐
│ Name     │ Value       │ Domain   │
├──────────┼─────────────┼──────────┤
│ wr_skey  │ abcdef123...│ weread...│
└──────────┴─────────────┴──────────┘
                        ↑
                    复制这个值
```

#### 步骤2：配置GitHub Secret

1. 打开GitHub仓库：https://github.com/xie-l/xie-l.github.io
2. 进入 **Settings → Secrets and variables → Actions**
3. 检查是否存在 **WEREAD_COOKIE**
   - **如果存在**：点击 **Update** 更新值
   - **如果不存在**：点击 **New repository secret**
4. **Name**: `WEREAD_COOKIE`
5. **Value**: 粘贴刚才复制的Cookie值
6. 点击 **Save** 或 **Add secret**

**截图说明**：
```
Settings → Secrets and variables → Actions
┌─────────────────────────────────────┐
│ Repository secrets                  │
│                                     │

│                                     │
│ [New repository secret]             │
└─────────────────────────────────────┘
```

#### 步骤3：触发GitHub Actions工作流

1. 访问：https://github.com/xie-l/xie-l.github.io/actions/workflows/fetch-weread.yml
2. 点击 **Run workflow** 按钮
3. 在弹出的下拉菜单中：

4. 等待运行完成（约1-2分钟）
