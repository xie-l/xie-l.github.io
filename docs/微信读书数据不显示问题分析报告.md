# 微信读书数据不显示问题分析报告

**报告日期**：2026年4月8日
**问题描述**：首页微信读书模块不显示
**分析结果**：功能正常，数据缺失导致

---

## 一、问题诊断

### 1.1 现象
- 首页侧边栏没有显示"正在阅读"模块
- 检查HTML发现DOM元素存在但设置了`display:none`

### 1.2 根本原因
**数据文件为空导致模块隐藏**

文件：`data/weread.json`
```json
{
  "reading": null,
  "recent": [],
  "updated": "2026-04-08 04:40 UTC"
}
```

### 1.3 代码逻辑分析

在`index.html`第1616行的JavaScript代码：

```javascript
if (!d.reading && (!d.recent || d.recent.length === 0)) return;
```

**逻辑说明**：
- 如果没有正在读的书 (`reading: null`)
- 且没有最近读过的书 (`recent: []`)
- 则直接返回，不显示widget

**这是正确的行为设计**，避免展示空内容。

---

## 二、为什么数据为空

### 2.1 数据抓取机制

数据来源：GitHub Actions自动抓取
- 工作流：`.github/workflows/fetch-weread.yml`
- 执行频率：每天 02:05 UTC（北京时间 10:05）
- 需要配置：`WEREAD_COOKIE`（微信读书Cookie）

### 2.2 可能的原因

1. **Cookie未配置**
   - GitHub仓库中未设置`WEREAD_COOKIE` Secret
   - 导致抓取脚本写入空数据

2. **Cookie已过期**
   - 微信读书Cookie有效期7-30天
   - 过期后需要重新获取并更新

3. **抓取失败**
   - 网络问题或API变更
   - 脚本错误

---

## 三、修复方案

### 方案A：配置微信读书Cookie（推荐，长期有效）

#### 步骤1：获取Cookie

1. 打开浏览器，访问 https://weread.qq.com
2. 扫码登录微信读书
3. 按F12打开开发者工具 → Application → Cookies → weread.qq.com
4. 找到 `wr_skey` Cookie，复制其值
5. 或者复制整个Cookie字符串

#### 步骤2：配置GitHub Secret

1. 打开GitHub仓库：https://github.com/xie-l/xie-l.github.io
2. 进入 Settings → Secrets and variables → Actions
3. 点击 "New repository secret"
4. Name: `WEREAD_COOKIE`
5. Value: 粘贴Cookie值
6. 点击 "Add secret"

#### 步骤3：触发工作流

1. 进入 Actions → Fetch WeChat Reading Data
2. 点击 "Run workflow" → 选择main分支
3. 等待执行完成（约1-2分钟）
4. 检查`data/weread.json`是否更新

#### 步骤4：验证显示

1. 刷新首页
2. 检查侧边栏是否显示"正在阅读"模块
3. 检查数据是否正确

## 五、预防措施

### 5.1 Cookie过期提醒

在GitHub Actions中添加提醒机制：

```yaml
# 在fetch-weread.yml中添加
- name: Check cookie expiration
  if: failure()
  run: |
    echo "::error::微信读书Cookie可能已过期，请重新获取并更新WEREAD_COOKIE"
```

### 5.2 监控数据更新

添加数据新鲜度检查：

```javascript
// 在loadWeRead函数中添加
const updated = new Date(d.updated);
const daysSinceUpdate = (Date.now() - updated) / (1000 * 60 * 60 * 24);
if (daysSinceUpdate > 7) {
    console.warn('微信读书数据超过7天未更新');
}
```

### 5.3 手动触发备份

当自动抓取失败时，可以手动运行工作流：

1. 进入 Actions → Fetch WeChat Reading Data
2. 点击 "Run workflow"
3. 选择分支并运行
