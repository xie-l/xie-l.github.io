# GitHub API 管理功能使用说明

## 🎉 功能介绍

现在你的个人主页集成了完整的GitHub API管理功能，支持：

- ✅ **在线撰写博客** - 无需手动创建HTML文件
- ✅ **文件上传** - 拖拽或选择文件上传
- ✅ **自动保存草稿** - 防止意外丢失内容
- ✅ **文件管理** - 查看和管理已上传的文件
- ✅ **纯前端实现** - 无需后端服务器

---

## 🚀 快速开始

### 1. 获取GitHub Personal Access Token

访问 https://github.com/settings/tokens

1. 点击 "Generate new token" (classic)
2. 填写 Note（例如：个人主页管理）
3. 选择过期时间（建议选择 90 days）
4. **重要**：勾选以下权限：
   - ✅ `repo` - 访问仓库
   - ✅ `workflow` - 更新GitHub Actions工作流文件
5. 点击 "Generate token"
6. **复制生成的token**（只显示一次）

### 2. 访问管理后台

访问：`https://xie-l.github.io/admin/login.html`

### 3. 登录

- **Token**：粘贴你复制的GitHub Personal Access Token
- **仓库名称**：自动填充为 `xie-l.github.io`
- 点击 "登录"

### 4. 开始管理

登录成功后，会自动跳转到管理后台（dashboard.html）

---

## 📝 撰写博客

### 创建新文章

1. 在管理后台点击 "撰写博客"
2. 填写信息：
   - **文章标题** - 必填
   - **文章分类** - 选择"技术思考"或"生活日记"
   - **标签** - 用逗号分隔，例如：`机器学习, Python, 科研`
   - **文章内容** - 支持纯文本，会自动转换为HTML

3. 操作：
   - **预览** - 在新窗口预览文章效果
   - **保存草稿** - 保存到浏览器本地存储
   - **发布文章** - 上传到GitHub仓库

### 自动保存

系统会自动保存草稿到浏览器的localStorage，下次打开时会自动加载。

### 生成的文件

发布后，会在以下目录生成HTML文件：
- 技术思考：`blog/tech/YYYY-MM-DD-文章标题.html`
- 生活日记：`blog/life/YYYY-MM-DD-文章标题.html`

## 📁 上传文件

### 上传方式

1. **点击上传区域** - 选择文件
2. **拖拽文件** - 拖拽到上传区域
3. **支持多文件** - 可同时选择多个文件

### 文件限制

- **单个文件大小**：最大10MB
- **支持类型**：图片（jpg, png, gif）、文档（pdf, doc, docx）、压缩包（zip, rar）等

### 上传目录

可选择上传到：
- **照片相册** - `files/photos/`
- **文档资料** - `files/documents/`
- **博客图片** - `blog/images/`

### 文件管理

上传后，文件会显示在列表中，包含：
- 文件图标
- 文件名
- 文件大小
- 上传进度条
- 删除按钮

## 🔐 安全说明

### Token安全

1. **Token存储**：使用浏览器的localStorage存储，只在本地保存
2. **Token显示**：在设置页面只显示前10个字符，其余用...隐藏
3. **HTTPS**：确保通过HTTPS访问，防止Token被窃取
4. **定期更换**：建议定期更换GitHub Personal Access Token

### 权限控制

Token只需要以下权限：
- `repo` - 读写仓库内容
- `workflow` - 更新工作流文件（可选，用于触发GitHub Pages部署）

**不要**给Token过多权限，遵循最小权限原则。

## 🎯 使用技巧

### 1. 撰写博客技巧

- **标题**：简洁明了，反映文章内容
- **标签**：添加相关标签，方便分类
- **内容**：分段清晰，逻辑连贯
- **预览**：发布前务必预览效果

### 2. 文件上传技巧

- **命名规范**：使用有意义的文件名，避免中文和特殊字符
- **分类存放**：根据文件类型选择正确的目录
- **批量上传**：可以一次选择多个文件上传
- **大文件**：超过10MB的文件建议压缩后再上传

### 3. 故障排查

**登录失败**：
- 检查Token是否正确
- 确认Token未过期
- 检查Token权限是否足够

**上传失败**：
- 检查文件大小是否超过限制
- 确认网络连接正常
- 查看浏览器控制台错误信息

**发布失败**：
- 确认仓库名称正确
- 检查是否有同名文件
- 查看GitHub API限制

## 📊 技术实现

### 纯前端架构

```
浏览器
    ↓
HTML/CSS/JS
    ↓
GitHub API (REST)
    ↓
GitHub仓库
    ↓
GitHub Pages
```

### 核心功能

1. **身份验证**
   - 使用GitHub Personal Access Token
   - HTTP Basic Authentication

2. **文件操作**
   - 创建/更新：PUT /repos/{owner}/{repo}/contents/{path}
   - 读取：GET /repos/{owner}/{repo}/contents/{path}
   - 删除：DELETE /repos/{owner}/{repo}/contents/{path}

3. **内容处理**
   - Base64编码/解码
   - Markdown转HTML
   - 自动生成HTML结构

### 浏览器兼容性

- ✅ Chrome 80+
- ✅ Firefox 75+
- ✅ Safari 13+
- ✅ Edge 80+

## 📞 常见问题

### Q: Token过期了怎么办？

A: 重新生成Token，在登录页面重新登录。

### Q: 可以管理多个仓库吗？

A: 目前只支持单个仓库，可以修改代码支持多仓库。

### Q: 文件上传有数量限制吗？

A: GitHub单个文件限制100MB，建议每个文件不超过10MB。

### Q: 发布文章后多久能看到？

A: GitHub Pages通常需要1-2分钟部署，清除缓存后刷新即可。

### Q: 如何删除已发布的文章？

A: 目前需要在GitHub网站上手动删除，后续可以添加删除功能。
